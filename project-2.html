Project 2
Phone Unlock - Touch & Sensor(s) Based

Everyone carries a phone in today’s world. Our phones hold with them information about us that we need. In many cases our phones know us better than we know ourselves. Therefore, it is important to keep our devices secured so that we are the only ones who can unlock them.

Traditional methods of unlocking phones with passwords and bio detectors like face and finger scans have kept us safe. However, these methods of security could use a creative update!

Our group wants to introduce two new ways to unlock our phones. The first unlocking technique is based on gesture input: as patterns light up buttons the user must watch for their secret pattern to be displayed and then repeat that pattern correctly to unlock. The second unlock technique is based on phone sensor data: as the user must interact with the phone’s camera as another person’s hand and act out their secret handshake correctly to unlock..

10 Sketches for Clarity

1. This is an optimistic idea for the random password generator idea. Upon pressing the “LOCK ME!” on the introduction scree, user is then taken to the password generation screen where upon pressing of the “ROLL THE DICE” button located on the bottom right of the screen a random password is generated the field where the stars (asterisks) are present. The generated password may have a random length between 3 to 12 as well as a random selection of character between 0 to 9, A to Z, a to z and non-alphanumeric characters such as punctuation marks and backets. The bottom right is a button named “EXPORT” with the purpose of exporting the generated password.

2. This is a relatively more realistic idea compared to the previous where the user may select the complexity of the password using the three buttons on the top “EASY”, “MEDIUM” or “HARD”. Upon pressing the “GENERATE PASSWORD”, a random password is generated. With each difficulty more variations and lengths are included which consist of alphanumeric and non-alphanumeric characters. Upon generation of each password, its respective stats are displayed at the bottom of the screen.

3. This random password generator is an input randomizer. It takes three strings of alphanumeric and or non-alphanumeric characters, randomizes characters from all three strings and upon pressing of the “GENERATE” button, a password of random length is generated and displayed below

4. A random password generator that provides more control to the user where the user has options to select from which category should the password include characters from such as alphabets, numbers, special characters, upper case, lower case as well as the option to choose the length of character of the generated password. Upon pressing of the “GENERATE” button, the random password is generated based on the features selected.

5. A more of an interactive password generator where there a random set of characters appears on screen recursively for 12 times. Upon each iteration, only one of the displayed characters may be selected. Upon the completion of the 12th iteration of the random sequence, the selected characters from the 1st iteration to the 12th iteration are concatenated as a string sequence which results in the generation of the random password. For succession from the present iteration to the next, the arrow button at the bottom right of the screen is to be pressed.

6. A potentially game like password generator where upon pressing of the “SHOOT ME” button, a ball is launched between the constantly rotating angle between the specified two lines marked by the arrow heads. Once the balls is launched, the “SHOOT” me button is replaced by a random character. Each side of the octagon has 1 random character and once touched, is replaced by another random character. The ball goes through a damped harmonic motion through collision and each time the ball touches a side, that character is recorded until the ball stops moving. Thus, the sequence of all characters that the ball touched becomes a string resulting in the random generated password.

7. A unique password generator where screen is full of random coloured circles with random sizes. Only 1 circle is to be selected and upon pressing the “BURST US !!” button, the spelling of selected colour is replaced with visually similar looking character as Aqua with @Qu@ and the resultant is the generated password.

8. Takes a pre-existing password, changes random upper to lower case and or vice versa, replacing alphabets with visually similar character, adds K or Y2K denoting the 1000s or 2000s series and generates a random password by pressing the “RENOVATE ME” button.

9. A technical password generator where the complexity and strength of the generated password is based on the amount of CPU and RAM percentage allocated. The random password is generated by pressing the “KABOOM” button.

10. Takes an input of two alphanumeric strings, replaces characters with similarly looking random characters, randomizes the order of characters and concatenated the 2 resulting strings into a single which becomes the random generated password. The process is triggered by Pressing the “MEMORY ANESIA” text-like button.

11. A password generator that might classify as an extremely basic machine learning based password generator where each succession of the password generation increases the complexity of the generated password by at least 5%. 


In the end, our group decided to collaborate on a sketch of to combine our ideas. Our combination of ideas resulted in sketch number 11. Sketch number 11 is a password generator that allows a user to increase the length of the password by repeatedly clicking the generate password button and customize the password by checking off the options to include upper letters, numbers, and special characters. So, when an a user is angry repeated button pressing will be a great outlet for emotions while solving the issue with a new password!

The idea is with more interaction with the button the stronger and more complex the password becomes, thus providing a great first line of defense against unauthorized access. The stronger the password, the more protected one’s computer will be from malicious entities. We ended up choosing this because the idea was a unique spin on the traditional password generator that provided the same real-life utility at the same time as being a creative use of button interaction. For the technical components, the project with constructed with best practice use of JavaScript, CSS, and HTML. The team used all aspects in combination to create the working demo.
